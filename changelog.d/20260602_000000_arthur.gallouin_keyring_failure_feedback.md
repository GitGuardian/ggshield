### Fixed

- ggshield no longer silently leaves API tokens in cleartext when it cannot write them to the OS credential store. This happened on macOS (error `-25244`) when the ggshield binary path changed — for example after a `mise`/`asdf` reshim, a `pyenv`/`pipx` reinstall or a Homebrew upgrade — because the Keychain entry's ACL was tied to the previous path. A clear warning is now shown explaining that the token was stored in plaintext and how to recover.

### Added

- New `ggshield auth status` command that diagnoses how API tokens are stored: it reports the credential-store backend (macOS Keychain, Windows Credential Locker, Linux Secret Service), whether it is reachable, and where each instance's token actually lives (credential store vs cleartext config file), with a copy-pasteable fix when a token unexpectedly stays in plaintext.
- A confirmation message is now shown when a cleartext token is migrated to the OS credential store.
